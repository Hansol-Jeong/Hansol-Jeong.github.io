JFC(1)

자바 컬렉션 프레임워크 (Java Collections Framework; JCF)

컬렉션 프레임워크란

  • java.utils에 속한 일련의 클래스로, 자료구조를 담당

  • 잘 짜여진 인터페이스를 기반으로 다양한 자료구조를 구현

  • 제네릭 클래스로 되어 있어, 다양한 객체를 요소로 담을 수 있다.

    JCF 인터페이스

  • List 인터페이스

    • Collection 인터페이스를 상속
    • 순서가 있는 데이터의 집합으로, 데이터의 중복을 허용
    • 구현체: ArrayList, LinkedList(, Vector, Stack)

  • Set 인터페이스

    • Collection 인터페이스를 상속
    • 순서가 없는 데이터의 집합으로, 데이터의 중복을 허용하지 않음
    • 구현체: HashSet, TreeSet

  • Map 인터페이스

    • Key-Value 쌍으로 이루어진 데이터의 집합
    • 구현체: HashMap, TreeMap, Hashtable, Properties

      Collection 인터페이스

  • Collection 인터페이스의 주요 메소드

    • add( E e) : 요소 e를 컬렉션에 추가한다.
    • addAll(Collection<? extends E c> : 다른 컬렉션 c의 모든 요소를 컬렉션에 추가한다
      도전 과제: 와일드 카드란
    • clear() : 컬렉션의 모든 요소를 제거한다.
    • contains(Object o) : 컬렉션에 요소 o가 존재하는지 확인한다.
    • containsAll(Collection<?> c) 컬렉션 c의 모든 요소가 컬렉션에 포함되는 지 확인한다.
    • equals(Collection<?> c) : 컬렉션 c와 같은 내용을 포함하는지 비교한다.
    • isEmpty() : 컬렉션이 비어있는지를 확인한다.
    • iterator() : 컬렉션의 Iterator을 반환한다.
    • remove(Object o) : 컬렉션에서 요소 o를 제거한다.
    • removeAll(Collection<?> c) : 컬렉션 c에 속한 모든 요소를 컬렉션에서 제거한다.
    • retainAll(Collection<?> c) : 컬렉션 c에 포함된 객체만 남기고 나머지 요소를 제거한다.

  • Iterator

    • iterator()로 반환되는 객체로, Collection에 저장된 요소에 접근하기 위해 사용

      • 메소드/설명

        • hasNext(): 다음 요소가 있는지 확인

        • next(): 다음 요소가 있을 경우 반환

        • remove(): 현재 위치의 요소를 삭제

          • Iterator의 활용

          Iterator iter = collection.iterator();
          while(iter.hasNext()) {
          String string = iter.next();
          System.out.println(string); }
          이것을 간단하게 하면

        for(String string: collection) {
        System.out.println(string); }
