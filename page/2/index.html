Day_1

    백엔드 개발자도 프론트엔드를 어느정도는 배워놔야 한다-김데레사

HTML_CSS

    오늘부터 5일동안 김 데레사 선생님과 함께 해당 웹 페이지를 만들어 볼 것인데요 다 만들지는 않습니다. 해당 페이지를 클론코딩하면서 HTML과 CSS대한 이해를 키우고 자연스럽게 프론트엔드 팀과 협업하기 위한 기틀을 닦을 것입니다.

설계 순서

  • 논리적인 순서
  • 시멘틱 마크업
  • 네이밍
  • 구조 설계

논리적인 순서

     많은 프론트엔드 개발자들이 실수하는 것 중에 하나는 웹을 논리적인 순서로 구현하지 않고 디자인 순서로 구현한 다는 것입니다. 예를 하나 들겠습니다.

    해당 페이지는 네이버의 로그인 화면입니다. 디자인 순서로 보면 아이디, 비밀번호를 친 후 로그인버튼과 그 아래 로그인 상태 유지가 나옵니다. 이것을 코드로 그대로 옮긴다면 구현 순서 또한 똑같을 것입니다. 하지만 그러면 안된다는 것입니다.

    이것은 같은 페이지의 CSS를 걷어낸 네이버의 로그인 화면입니다. 역시 아이디, 비밀번호, 로그인버튼, 그다음 로그인 상태유지가 있음을 볼 수 있습니다. 이럴 때 우리는 논리적으로 생각하면서 웹 페이지를 만들어야 한다고 하셨습니다.
“로그인 누르기 전에 로그인 상태를 유지 할지 말지 선택하는게 논리적으로 맞다” 라고 데레사 선생님께서는 말씀하셨습니다. 때문에 디자인 시안만 받아보고 그대로 코딩하면 이런결과가 나온다고 하셨습니다. 물론 이것은 매우 사소해서 아무도 신경쓰지 않겠지만 개발자또한 사소하게 생각해서 대충대충 넘어가면 안된다는 의미인것입니다.

시멘틱 마크업

     또 많은 프론트엔드 개발자들이 저지르는 실수 중 하나는, 현업에 들어가면 대부분 div태그 하나로 모든것을 해결하려 한다는 것입니다. 이는 매우 잘못된 생각이며, 모든 태그는 용도에 맞게 쓸 때 가장 아름답다고 하셨습니다.

Create Database

use jbs;
CREATE TABLE user1(
user_id INT,
name varchar(20),
email varchar(20),
age INT(3),
rDate DATE
);

  • jbs데이터 베이스에 user1테이블을 생성하고 그 안에는 user_id, name, email, age, rDate가 각각의 데이터 형식으로 포함되어 있습니다.

    INSERT INTO user1 (user_id, name, email, age, rDate)
    values(1, “Andy”, “Andy@gmail.com“, 23, “2020-10-01”);
    SELECT * FROM user1;
    SHOW TABLES;
    DESC user1;

  • 데이터를 주입합니다.

    CREATE TABLE user2 (
    user_id INT PRIMARY KEY AUTO_INCREMENT,
    name varchar(20) NOT NULL,
    email varchar(20) unique NOT NULL,
    age INT(3) DEFAULT 30,
    rDate TIMESTAMP
    );

    • user2테이블을 생성합니다. 이 때
    • Primary key: 한 개 혹은 여러 개의 칼럼으로 테이블 내의 각 행들을 구별하기 위한 목적을 갖고 있다. PK는 NULL 값을 갖을 수 없고(NOT NULL), Unique key라는 특징을 갖게 된다.
    • Auto_Increment : 데이터가 증가할 때마다 인덱스가 1씩 늘어난다.

  • 때문에 user_id와 age, rDate 3가지는 초기값이 있다. name과 email이 null이면 안되기 때문에 이 두 값을 넣어줘야 한다

Select-From

선택합니다(Select) 특정 컬럼들을(Column) 특정 테이블로부터(From)

use world;
Select * from country;

  • world database의 country 테이블 로부터 모든(*) column을 선택하여 출력합니다.

    Where

    From절에 조건을 하나 겁니다.

    SELECT code, name, population
    FROM country
    WHERE population>=200000000;

  • (world)country 테이블 로부터 code, name, population column을 선택하여 출력합니다. 이 때 population >= 200000000 인 행만 출력하도록 합니다.

    And

    조건을 연달아 추가 할 수 있습니다.

    SELECT code, name, population
    FROM country
    WHERE population >=200000000 AND population <=300000000;

    • country 테이블로부터 code, name, population 열들을 출력하고 이 때 200000000<population< 300000000 입니다.

      Between

      부등호를 쓰지 않고 범위를 표현할 수 있습니다.
  SELECT code, name, population
    FROM country
    WHERE population BETWEEN 200000000 AND 300000000;

  • country 테이블로부터 code, name, population 열들을 출력하고 이 때 200000000<population< 300000000 입니다.

    특정 column의 특정 데이터를 특정 할 수 있습니다.

    SELECT code, name, continent, population
    FROM country
    WHERE (continent = “Asia” or continent = “Africa”) AND population >=100000000;

    • continent = “Aisa” 또는 continent = “Africa” 이고 population >100000000 인 code, name, continent, population 으로 이루어진 행들을 country 테이블로부터 뽑아낼 수 있습니다.

      Not In

      특정 조건을 포함하지 않도록 할 수 있습니다.

    SELECT code, name, continent, population
    FROM country
    WHERE continent NOT IN (“Asia”, “Africa”) AND population >=100000000;

    • continent column에 Asia, Africa가 포함되지 않고, population>100000000인 결과를 출력합니다.

      정규표현식

      정규표현식을 사용하여 데이터를 추출 할 수 있습니다.

    SELECT code, name, GovernmentForm
    FROM country
    WHERE GovernmentForm LIKE “%Republic%”;

    • GovernmentForm column에 Republic이 포함되는 결과를 출력합니다.

      Order By

      특정 조건대로 정렬시킬 수 있습니다.

    SELECT code, name, population
    FROM country
    ORDER BY population ASC;

    • ASC: 오름차순으로 정렬합니다. 초깃값이기 때문에 따로 쓰지 않더라도 적용됩니다.

      SELECT code, name, population
      FROM country
      ORDER BY population DESC;

  • DESC: 내림차순으로 정렬합니다.

Order By 에 조건을 2개이상 넣을 수 있습니다.

SELECT countryCode, name, population
FROM city
WHERE countrycode IN ("USA", "KOR", "JPN")
ORDER BY countrycode ASC, population DESC;

  • countryCode, name, population column을 city테이블에서 가져 올 때 countrycode = “USA”, “KOR”, “JPN”인 행들만 포함합니다. 이 때 countrycode를 기준으로 오름차순으로 정렬하며, countrycode, 가 같다면 population을 기준으로 내림차순으로 재 정렬합니다.

    LIMIT

    출력 범위를 지정할 수 있습니다.

    SELECT countrycode, name, population
    FROM city
    WHERE countrycode IN (“USA”, “KOR”, “JPN”)
    ORDER BY countrycode ASC, population DESC
    LIMIT 5,2; #6,7위

  • Limit 5, 2: 첫 번째 결과부터 5개의 결과를 스킵하고, 2개의 결과를 출력합니다.

COUNT

Group By와 함께 사용되는 함수이며, Group By의 조건으로 묶였을 때 중복을 세어주는 함수

SELECT countrycode, count(countrycode)
FROM city
GROUP BY countrycode;
  • city 테이블에서 countrycode, count(countrycode) column을 출력한다. 이 때 Group By로 차트가 묶인 상태이기 때문에 countrycode는 중복되지 않는 값이 출력되며 이 때 count는 countrycode가 몇개 묶였는지 나타낸다.

MAX

Group By와 함께 쓰이며, 특정 조건의 최대값을 출력할 수 있습니다.

SELECT continent, MAX(GNP), MAX(population)
FROM country
GROUP BY continent;
  • continent 를 기준으로 continent가 중복된다면 해당 데이터들을 묶고, 이 때 묶인 데이터 들 중 최대 GNP와 최대 population을 출력합니다.

SUM

Group By와 함께 쓰이며, 데이터가 Group By에 의해 묶일 때 그 값들의 총합을 출력합니다.

SELECT continent, SUM(GNP) AS S_GNP, SUM(population) AS S_population, SUM(GNP)/SUM(population) AS per 
FROM country
GROUP BY continent;

  • continent 를 기준으로 데이터를 묶고 이 때 GNP와 population을 모두 더해 각각 S_GNP와 S_population으로 나타냅니다.

    AVG

    Group By와 함께 쓰이며, 데이터의 평균을 나타낼 수 있습니다.

    SELECT continent, AVG(population), AVG(GNP), SUM(population) AS population
    FROM country
    WHERE population !=0 AND GNP !=0
    GROUP BY continent
    ORDER BY population DESC;

    • continent를 기준으로 묶은 데이터들 중 population != 0, GNP !=0인 행들을 출력하고 이 때 데이터는 population을 기준으로 내림차순으로 정렬하며 출력하는 column의 종류는 continent, population의 평균, GNP의 평균, population 의 총합 입니다.

HAVING

GROUP BY로 출력되는 결과를 필터링할 때 사용 합니다.

SELECT continent, SUM(population) AS S_POP
FROM country
GROUP BY continent
HAVING S_POP>=500000000
ORDER BY S_POP DESC;
  • continent로 묶은 데이터 들 중에서 S_POP:SUM(population)이 >500000000인 행들을 내림차순으로 출력합니다.

JFC(4)

Map 인터페이스

  • Key와 Value의 쌍으로 구성된 자료의 집합을 다루는 인터페이스

    • Key, Value 쌍은 Map.Entry<K, V> 객체로 관리

  • Key는 중복될 수 없으며, Value는 중복이 가능

  • Map<K, V>의 구현체

    • HashMap<K, V> : 가장 많이 사용되는 Map의 구현체
    • TreeMap<K, V> : 정렬 기준이 더해진 NavigableMap을 구현한 구현체
    • Hashtable<K, V> : HashMap과 유사하지만 성능이 떨어지는 대신 멀티스레드에 안전한 구현체
    • Properties : 속성 파일을 손쉽게 관리하는, Hashtable<K, V>의 하위 클래스 구현체
      @보통 동기화를 지원하면 느려진다@

Map<K, V>의 주요 메소드
메소드/설명

  • clear(): Map의 모든 요소를 삭제
  • containKey( Object Key): key가 존재하는 지 여부를 반환
  • containsValue(Object value): value가 존재하는지 여부를 반환
  • entrySet() : Map의 Entry요소를 Set에 담아 반환
  • get(Object key): key 에 해당하는 value객체 반환
  • keySet(): key요소를 Set형태로 반환
  • put(K key, V value): key와 value쌍으로 구성된 entry추가. 기존에 등록된 key라면 해당 value를, 없으면 null을 반환
  • putAll(Map<? extends K, ? extends V> m): m에 속한 모든 Entry추가
  • remove(Object key): key에 해당하는 Entry를 제거하고 반환, 등록되지 않은 key라면 null반환
  • values(): value객체들을 Collection에 담아 반환
  • size(): Map에 포함된 Entry객체의 수 반환
  • isEmpty(): Map에 데이터가 없는지 여부 반환

TreeMap<K, V>의 주요 메소드
메소드/설명

  • firstEntry(): 정렬된 첫 Entry반환
  • lastEntry(): 정렬된 마지막 Entry반환
  • lowerEntry(K key): key 바로 이전 Entry반환
  • higherEntry(K key): key바로 다음 Entry반환
  • floorEntry(K key) : key 또는 key보다 이전 Entry반환
  • ceilingEntry(K key) : key 또는 key보다 이후 Entry반환
  • pollFirstEntry(): 정렬된 첫 Entry를 제거하고 반환
  • pollLastEntry() : 정렬된 마지막 Entry를 제거하고 반환
  • descendingKeySet(): 내림차순으로 정렬하는 Set반환
  • descendingKeyMap(): 내림차순으로 정렬하는 Map반환
  • headMap(K toKey, boolean inclusive): toKey이전 Entry로 구성된 객체 반환
  • tailMap(K fromKey, boolean inclusive): fromKey이후 Entry로 구성된 객체 반환
  • subMap(K fromKey, boolean fromInclusive, k toKey, boolean toInclusive): fromKey 이후, toKey이전 Entry로 구성된 객체 반환

Properties의 주요 메소드

  • getProperty(): Key에 해당하는 속성 값을 반환
  • stringPropertyNames(): key의 목록을 Set으로 반환
  • setProperty(): key로 value를 등록하고 기존 value값을 반환
  • load(): reader를 이용해 속성을 읽어온다. (properties)
  • loadFromXML() :XML형식으로 저장된 속성을 읽어온다.
  • store(): writer를 이용해 속성을 저장한다.
  • storeToXML(): XML형식으로 속성파일을 저장한다.

Set 인터페이스

  • 순서가 없는 데이터의 집합을 다루는 인터페이스

  • 중복되는 데이터를 효율적으로 제거하는 데에 쓸 수 있음

  • Collection 인터페이스를 상속받아 메소드를 구현하고 있음

  • Set의 구현체

  • HashSet<E>

    • Set의 대표적인 구현체로, 기본적인 기능이 구현되어 있다.

  • TreeSet<E>

    • NavigableSet<E> 인터페이스를 구현하며, 이진 탐색 트리 자료구조이다.

    • 객체를 삽입하는 동시에 정렬되는 특성상 비교가 가능해야 한다.

        public TreeSet() // Comparable 구현체의 경우
        public TreeSet(Comparator<? super E> comparator) // 별도로 비교 객체 삽입

       - first() : 정렬된 첫 요소를 반환
       - last(): 정렬된 마지막 요소를 반환
       - lower(E e): e객체의 바로 이전 객체를 반환
       - higher(E e) : e 객체의 바로 다음 객체를 반환
       - floor(E e) : e 또는 e보다 이전 객체를 반환
       - ceiling(E e) : e 또는 e보다 이후 객체를 반환
       - pollFirst(): 정렬된 첫 요소를 제거하고 반환
       - pollLast(): 정렬된 마지막 요소를 제거하고 반환
       - descendingSet() : 내림차순으로 정렬하는 객체를 반환
       - headSet(E toElement, boolean inclusive): toElement 이전 요소로 구성된 객체 반환
       - tailSet(E fromElement, boolean inclusive): fromElement 이후 요소로 구성된 객체 반환
       - subSet(E fromElement, boolean fromInclusive, E toElement, boolean toInclusive): fromElement 이후, toElement 이전 요소로 구성된 객체 반환
