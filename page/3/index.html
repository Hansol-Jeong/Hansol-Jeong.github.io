Day_1

Select-From

선택합니다(Select) 특정 컬럼들을(Column) 특정 테이블로부터(From)

use world;
Select * from country;

  • world database의 country 테이블 로부터 모든(*) column을 선택하여 출력합니다.

    Where

    From절에 조건을 하나 겁니다.

    SELECT code, name, population
    FROM country
    WHERE population>=200000000;

  • (world)country 테이블 로부터 code, name, population column을 선택하여 출력합니다. 이 때 population >= 200000000 인 행만 출력하도록 합니다.

    And

    조건을 연달아 추가 할 수 있습니다.

    SELECT code, name, population
    FROM country
    WHERE population >=200000000 AND population <=300000000;

    • country 테이블로부터 code, name, population 열들을 출력하고 이 때 200000000<population< 300000000 입니다.

      Between

      부등호를 쓰지 않고 범위를 표현할 수 있습니다.
  SELECT code, name, population
    FROM country
    WHERE population BETWEEN 200000000 AND 300000000;

  • country 테이블로부터 code, name, population 열들을 출력하고 이 때 200000000<population< 300000000 입니다.

    특정 column의 특정 데이터를 특정 할 수 있습니다.

    SELECT code, name, continent, population
    FROM country
    WHERE (continent = “Asia” or continent = “Africa”) AND population >=100000000;

    • continent = “Aisa” 또는 continent = “Africa” 이고 population >100000000 인 code, name, continent, population 으로 이루어진 행들을 country 테이블로부터 뽑아낼 수 있습니다.

      Not In

      특정 조건을 포함하지 않도록 할 수 있습니다.

    SELECT code, name, continent, population
    FROM country
    WHERE continent NOT IN (“Asia”, “Africa”) AND population >=100000000;

    • continent column에 Asia, Africa가 포함되지 않고, population>100000000인 결과를 출력합니다.

      정규표현식

      정규표현식을 사용하여 데이터를 추출 할 수 있습니다.

    SELECT code, name, GovernmentForm
    FROM country
    WHERE GovernmentForm LIKE “%Republic%”;

    • GovernmentForm column에 Republic이 포함되는 결과를 출력합니다.

      Order By

      특정 조건대로 정렬시킬 수 있습니다.

    SELECT code, name, population
    FROM country
    ORDER BY population ASC;

    • ASC: 오름차순으로 정렬합니다. 초깃값이기 때문에 따로 쓰지 않더라도 적용됩니다.

      SELECT code, name, population
      FROM country
      ORDER BY population DESC;

  • DESC: 내림차순으로 정렬합니다.

Order By 에 조건을 2개이상 넣을 수 있습니다.

SELECT countryCode, name, population
FROM city
WHERE countrycode IN ("USA", "KOR", "JPN")
ORDER BY countrycode ASC, population DESC;

  • countryCode, name, population column을 city테이블에서 가져 올 때 countrycode = “USA”, “KOR”, “JPN”인 행들만 포함합니다. 이 때 countrycode를 기준으로 오름차순으로 정렬하며, countrycode, 가 같다면 population을 기준으로 내림차순으로 재 정렬합니다.

    LIMIT

    출력 범위를 지정할 수 있습니다.

    SELECT countrycode, name, population
    FROM city
    WHERE countrycode IN (“USA”, “KOR”, “JPN”)
    ORDER BY countrycode ASC, population DESC
    LIMIT 5,2; #6,7위

  • Limit 5, 2: 첫 번째 결과부터 5개의 결과를 스킵하고, 2개의 결과를 출력합니다.

#QUIZ 한국(KOR)의 경기도(KYONGGI)에 해당하는 도시 에서 인구가 많은 3개의 도시를 출력

SELECT countrycode, name, population
FROM city
WHERE countrycode = "KOR" AND DISTRICT = "KYONGGI"
ORDER BY population DESC
LIMIT 3;

#동북아시아에서 인당 GNP가 높은 5개 나라 출력

SELECT code, name, GNP, population, region, (GNP/population) AS GNP_PER_PERSON
FROM country
WHERE region = "Eastern Asia"
ORDER BY GNP_PER_PERSON DESC
LIMIT 5;

#인구밀도가 높은 국가 6~10위까지 출력하세요

SELECT code, name, population, surfaceArea, surfaceArea/population as Density
FROM country
ORDER BY Density DESC
LIMIT 5,5;

COUNT

Group By와 함께 사용되는 함수이며, Group By의 조건으로 묶였을 때 중복을 세어주는 함수

SELECT countrycode, count(countrycode)
FROM city
GROUP BY countrycode;
  • city 테이블에서 countrycode, count(countrycode) column을 출력한다. 이 때 Group By로 차트가 묶인 상태이기 때문에 countrycode는 중복되지 않는 값이 출력되며 이 때 count는 countrycode가 몇개 묶였는지 나타낸다.

MAX

Group By와 함께 쓰이며, 특정 조건의 최대값을 출력할 수 있습니다.

SELECT continent, MAX(GNP), MAX(population)
FROM country
GROUP BY continent;
  • continent 를 기준으로 continent가 중복된다면 해당 데이터들을 묶고, 이 때 묶인 데이터 들 중 최대 GNP와 최대 population을 출력합니다.

SUM

Group By와 함께 쓰이며, 데이터가 Group By에 의해 묶일 때 그 값들의 총합을 출력합니다.

SELECT continent, SUM(GNP) AS S_GNP, SUM(population) AS S_population, SUM(GNP)/SUM(population) AS per 
FROM country
GROUP BY continent;

  • continent 를 기준으로 데이터를 묶고 이 때 GNP와 population을 모두 더해 각각 S_GNP와 S_population으로 나타냅니다.

    AVG

    Group By와 함께 쓰이며, 데이터의 평균을 나타낼 수 있습니다.

    SELECT continent, AVG(population), AVG(GNP), SUM(population) AS population
    FROM country
    WHERE population !=0 AND GNP !=0
    GROUP BY continent
    ORDER BY population DESC;

    • continent를 기준으로 묶은 데이터들 중 population != 0, GNP !=0인 행들을 출력하고 이 때 데이터는 population을 기준으로 내림차순으로 정렬하며 출력하는 column의 종류는 continent, population의 평균, GNP의 평균, population 의 총합 입니다.

HAVING

GROUP BY로 출력되는 결과를 필터링할 때 사용 합니다.

SELECT continent, SUM(population) AS S_POP
FROM country
GROUP BY continent
HAVING S_POP>=500000000
ORDER BY S_POP DESC;
  • continent로 묶은 데이터 들 중에서 S_POP:SUM(population)이 >500000000인 행들을 내림차순으로 출력합니다.

    Quiz

1. 언어별 사용하는 국가의 수를 조회하고 많이 사용되는 언어 6위~ 8위 출력

SELECT language, count(CountryCode) AS count
FROM countrylanguage
GROUP BY language
ORDER BY count DESC
LIMIT 5,3;

2. 대륙별 나라의 갯수를 출력하고 국가가 많은 대륙 1위 ~3위까지 출력

SELECT continent, count(name) AS count
FROM country
GROUP BY continent
ORDER BY count DESC
LIMIT 3;

3. city테이블에서 국가 코드별 총인구를 출력, 총 인구순으로 내림차순 정렬, 총 인구가 5천만 이상인 국가코드만 출력

SELECT countrycode, SUM(population) AS S_pop
FROM city
GROUP BY countrycode
HAVING S_pop >=50000000
ORDER BY S_POP DESC;

IO

I/O

I/O와 스트림

  • I/O란 입출력(Input/Output)을 함께 부르는 것
  • 자바의 I/O는 스트림을 이용해 데이터를 주고 받는 형식으로 이루어짐
    • 데이터의 소스와 목적지를 노드(node)라 부름
    • 노드는 키보드, 모니터, 파일, 메모리, 데이터베이스, 다른 프로그램 등이 될 수 있음

노드 스트림

InputStream과 Reader

InputSTream의 주요 메소드 (byte 단위)

메소드/ 설명

  • read() : byte하나를 읽어서 int로 반환. 읽을 값이 없으면 -1

  • read(byte b[]) : 데이터를 읽어 b를 채우고, 읽은 바이트 수를 반환

  • read(byte b[], int offset, int len) : 최대 len개의 바이트를 읽어 b의 offset위치부터 채운다.

  • close() : 스트림을 종료하고 자원을 반납.

  • available() : 읽을 수 있는 데이터의 크기를 반납

  • skip(long n) : 스트림에서 n만큼 건너뛴다.

  • mark(int leadLimit) : reset() 으로 돌아갈 위치를 표시한다. readLimit은 돌릴 수 있는 최대 바이트 수

  • reset() : mark()가 호출된 지점으로 돌아간다.

  • markSupported() : mark, reset메소드의 지원 여부를 반환

Reader의 주요 메소드(Char 단위)
메소드/ 설명

  • int read() : char 하나를 읽어서 int로 반환. 읽을 값이 없으면 -1
  • int read(char cbuf[]) : 데이터를 읽어 cbuf를 채우고, 읽은 char수를 반환
  • int read(char cbuf[], int off, int len) : 최대 len개의 char를 읽어 cbuf의 offset 위치부터 채운다.
  • int read(java.nio.CharBuffer target) : NIO target에 데이터를 저장한다.
  • void close() : 스트림을 종료하고 자원을 반납
  • int available() : 읽을 수 있는 데이터의 크기를 반납
  • long skip(long n) : 스트림에서 n만큼 건너뛴다
  • void mark(int readAheadLimit) : reset()으로 돌아갈 위치를 표시한다. readLimit은 돌릴 수 있는 최대 바이트 수
  • void reset() : mark()가 호출된 지점으로 돌아간다
  • boolean markSupported() : mark, reset메소드의 지원 여부를 반환

OutputStream과 Writer

  • OutputStream의 주요 메소드(byte단위)

메소드/ 설명

  • write( int b): b내용을 byte로 출력
  • write(byte b[]) : b를 문자열로 변환하여 출력
  • write(byte b[], int off, int len) : b의 off부터 (off+len-1)만큼을 문자열로 변환하여 출력
  • close() : 스트림을 종료하고 자원을 반납. close() 내부적으로 flush()호출
  • flush() : 버퍼스트림에서 버퍼의 내용을 출력하고 비운다.

Writer의 주요 메소드(char단위)
메소드/ 설명

  • write(int c) : c내용을 char로 출력
  • write(char cbuf[]) : cbuf를 문자열로 변환하여 출력
  • write(char cbuf[], int off, int len) : cbuf의 off부터 (off+len-1)만큼을 문자열로 변환하여 출력
  • write(String str) : str을 출력한다
  • write(String str, int off, int len) : str의 off부터 (off+len-1)만큼을 출력
  • append(CharSequence csq) : csq를 출력하고 Writer반환
  • apend(char C) : c를 출력하고 Writer반환
  • close() : 스트림을 종료하고 자원을 반납. close() 내부적으로 flush()호출
  • flush() : 버퍼스트림에서 버퍼의 내용을 출력하고 비운다.

JFC(4)

Map 인터페이스

  • Key와 Value의 쌍으로 구성된 자료의 집합을 다루는 인터페이스

    • Key, Value 쌍은 Map.Entry<K, V> 객체로 관리

  • Key는 중복될 수 없으며, Value는 중복이 가능

  • Map<K, V>의 구현체

    • HashMap<K, V> : 가장 많이 사용되는 Map의 구현체
    • TreeMap<K, V> : 정렬 기준이 더해진 NavigableMap을 구현한 구현체
    • Hashtable<K, V> : HashMap과 유사하지만 성능이 떨어지는 대신 멀티스레드에 안전한 구현체
    • Properties : 속성 파일을 손쉽게 관리하는, Hashtable<K, V>의 하위 클래스 구현체
      @보통 동기화를 지원하면 느려진다@

Map<K, V>의 주요 메소드
메소드/설명

  • clear(): Map의 모든 요소를 삭제
  • containKey( Object Key): key가 존재하는 지 여부를 반환
  • containsValue(Object value): value가 존재하는지 여부를 반환
  • entrySet() : Map의 Entry요소를 Set에 담아 반환
  • get(Object key): key 에 해당하는 value객체 반환
  • keySet(): key요소를 Set형태로 반환
  • put(K key, V value): key와 value쌍으로 구성된 entry추가. 기존에 등록된 key라면 해당 value를, 없으면 null을 반환
  • putAll(Map<? extends K, ? extends V> m): m에 속한 모든 Entry추가
  • remove(Object key): key에 해당하는 Entry를 제거하고 반환, 등록되지 않은 key라면 null반환
  • values(): value객체들을 Collection에 담아 반환
  • size(): Map에 포함된 Entry객체의 수 반환
  • isEmpty(): Map에 데이터가 없는지 여부 반환

TreeMap<K, V>의 주요 메소드
메소드/설명

  • firstEntry(): 정렬된 첫 Entry반환
  • lastEntry(): 정렬된 마지막 Entry반환
  • lowerEntry(K key): key 바로 이전 Entry반환
  • higherEntry(K key): key바로 다음 Entry반환
  • floorEntry(K key) : key 또는 key보다 이전 Entry반환
  • ceilingEntry(K key) : key 또는 key보다 이후 Entry반환
  • pollFirstEntry(): 정렬된 첫 Entry를 제거하고 반환
  • pollLastEntry() : 정렬된 마지막 Entry를 제거하고 반환
  • descendingKeySet(): 내림차순으로 정렬하는 Set반환
  • descendingKeyMap(): 내림차순으로 정렬하는 Map반환
  • headMap(K toKey, boolean inclusive): toKey이전 Entry로 구성된 객체 반환
  • tailMap(K fromKey, boolean inclusive): fromKey이후 Entry로 구성된 객체 반환
  • subMap(K fromKey, boolean fromInclusive, k toKey, boolean toInclusive): fromKey 이후, toKey이전 Entry로 구성된 객체 반환

Properties의 주요 메소드

  • getProperty(): Key에 해당하는 속성 값을 반환
  • stringPropertyNames(): key의 목록을 Set으로 반환
  • setProperty(): key로 value를 등록하고 기존 value값을 반환
  • load(): reader를 이용해 속성을 읽어온다. (properties)
  • loadFromXML() :XML형식으로 저장된 속성을 읽어온다.
  • store(): writer를 이용해 속성을 저장한다.
  • storeToXML(): XML형식으로 속성파일을 저장한다.

Set 인터페이스

  • 순서가 없는 데이터의 집합을 다루는 인터페이스

  • 중복되는 데이터를 효율적으로 제거하는 데에 쓸 수 있음

  • Collection 인터페이스를 상속받아 메소드를 구현하고 있음

  • Set의 구현체

  • HashSet<E>

    • Set의 대표적인 구현체로, 기본적인 기능이 구현되어 있다.

  • TreeSet<E>

    • NavigableSet<E> 인터페이스를 구현하며, 이진 탐색 트리 자료구조이다.

    • 객체를 삽입하는 동시에 정렬되는 특성상 비교가 가능해야 한다.

        public TreeSet() // Comparable 구현체의 경우
        public TreeSet(Comparator<? super E> comparator) // 별도로 비교 객체 삽입

       - first() : 정렬된 첫 요소를 반환
       - last(): 정렬된 마지막 요소를 반환
       - lower(E e): e객체의 바로 이전 객체를 반환
       - higher(E e) : e 객체의 바로 다음 객체를 반환
       - floor(E e) : e 또는 e보다 이전 객체를 반환
       - ceiling(E e) : e 또는 e보다 이후 객체를 반환
       - pollFirst(): 정렬된 첫 요소를 제거하고 반환
       - pollLast(): 정렬된 마지막 요소를 제거하고 반환
       - descendingSet() : 내림차순으로 정렬하는 객체를 반환
       - headSet(E toElement, boolean inclusive): toElement 이전 요소로 구성된 객체 반환
       - tailSet(E fromElement, boolean inclusive): fromElement 이후 요소로 구성된 객체 반환
       - subSet(E fromElement, boolean fromInclusive, E toElement, boolean toInclusive): fromElement 이후, toElement 이전 요소로 구성된 객체 반환

List 인터페이스

  • 순서가 있는 데이터의 집합을 다루는 인터페이스

  • List는 인덱스를 이용해 요소를 구분할 수 있어, 데이터의 중복을 허용

  • Collection 인터페이스를 상속받았으며, 추가 메소드가 구현되어 있다.

메소드/설명

  • add(int index, E element) : index 위치에 요소 element를 삽입한다.

람다식 (Lambda Expression)

람다식이란

  • Java 1.8에서 추가된 함수형 프로그래밍 기법
  • 객체지향 프로그래밍과 다르게, 비즈니스 로직만을 빠르게 구현하는 특징
  • 객체지향 언어인 Java에서 메소드를 함수처럼 사용하는 형식

    람다식의 예

배열의 정렬

  • 기본 정렬 방식을 이용한 정렬

    String [] strings = {"fast", "campus", "java", "backend", "school"};
System.out.println(Arrays.toString(strings));
Arrays.sort(strings);
System.out.println(Arrays.toString(strings));

  • 익명 내부 클래스를 이용한 정렬 방식 변경

     Arrays.sort(strings, new Comparator<String>() { // 객체를 생성하여 전달
  @Override
  public int compare(String o1, String o2) {
      return o1.compareTo(o2) * -1; // 내림차순
  }
 });

  • 람다식을 이용한 정렬 방식의 변경

    Arrays.sort(strings, (o1, o2) -> {return o1.compareTo(o2) * -1;});
Comparator<String> comp = (o1, o2) -> {
  return o1.compareTo(o2) * -1;
};
Arrays.sort(strings, comp);

  • Comparator객체에서 어떤 함수가 return을 하는데? -> 그런 모호함을 없애기 위해 실행되는 함수는 하나여야 한다.ㅇ

  • 람다식을 쓰기 위해서는 추상메소드가 단 하나 존재해야 합니다. 그것을 검사하는 방법은

함수형 인터페이스 (Functional Interface)

  • 추상 메소드가 단 하나 존재하는지 검사
    @FunctionalInterface
    public interface Comparator<T> {
    int compare(T o1, T o2);
    ...
    }

람다식의 형식

(String x) -> { System.out.println(x); }

(x) -> { System.out.println(x); }

x -> System.out.println(x)

() -> System.out.println("x");  // () 생략 불가능

(x) -> {
    System.out.println(x);
    return x;
}

(x) -> "x: " + x

람다식의 변수 참조

  • 익명 내부 클래스와 동일한 규칙으로 변수 참조

  • 문법적으로 this의 사용법만 다름

    • 익명의 내부 클래스와 달리 외부 클래스를 참조
      • 익명의 내부 클래스의 this는 익명의 내부클래스를 가르키고, 람다식의 this는 외부 클래스를 가리킨다.
        @FunctionalInterface
        interface FunctionalInterface {
        String method();
        }
        class Main {
        void functionalMethod(FunctionalInterface obj) {
            System.out.println(obj.method());
        }
        public static void main(String[] args) {
            functionalMethod(()-> {
                System.out.println("this: " + this);
                System.out.println("OuterClass.this: " + Main.this);
                return "Lambda expression used."
            });
            functionalMethod(new FunctionalInterface() {
                @Override
                String method() {
                    System.out.println("this: " + this);
                    System.out.println("OuterClass.this: " + Main.this);
                    return "Anonymous inner class used."
                });
        }
        }

        IntUnaryOperator multTwo = x -> x * 2;

Regular_Expression

정규표현식 (Regular Expression)

정규표현식

  • 특정 조건에 맞는 문자열을 검색/치환하는 패턴 기반의 식

  • 이메일, 전화번호 등 특정한 형식에 맞는지 validation하는 데에 사용

    정규표현식 표현 방법

  • 정규 표현식의 기본 작성 방식

    “/패턴/[플래그(Opt.)]”
    기본 메타 문자

  • 메타 문자/ 의미

    • ^x : x로 시작
    • x$ : x로 끝
    • .x : x앞에 하나의 문자가 있음
    • x+ : x가 1번 이상 반복
    • x? : x가 있거나 없음
    • x* : x가 0번 이상 반복
    • x|y : x 또는 y가 있음

  • 괄호의 활용

    • (xy) : 괄호 안의 내용을 그룹화
    • x{n} : n번 반복됨
    • x{n, } : n번 이상 반복됨
    • x{n, m) : n번 이상, m번 이하 반복됨
    • [xy] : x 또는 y
    • [a-z] : 알파벳 소문자 a~z
    • [0-9]: 숫자
    • [가-힣] : 한글
    • [^x] : x가 아닌 것
    • [^0-9] : 숫자가 아닌 것

  • 축약 문자

    • ^, . , … : \뒤에 나오는 문자를 문자로 처리
    • b : 단어의 경계를 찾는다.
      \B : 단어의 경계가 아닌 것을 찾는다
      \d : 숫자를 찾는다
      \D : 숫자가 아닌 것을 찾는다.
      \s : 공백 문자를 찾는다.
      \S : 공백 문자가 아닌 것을 찾는다.
      \w : [a-zA-z0-9_]
      \W : [^a-zA-Z0-9_]

  • 유용한 정규식 예

  • 한글 이름 : ^[가-힣]{2,5}$ : 한글로 시작하고 2글자이상 5글자 이하
  • 핸드폰 번호: ^01(0|1|2|6|9)[-\s]?\d{3,4}[-\s]?\d{4}$
  • 이메일 주소: ^[\w\.-]{1,64}@[\w\.-]{1,252}\.\w{2,4}$

    Flag

Flag/ 기능

  • g : 문자열 내 모든 패턴을 찾음
  • i : 대소문자를 구분하지 않음
  • m : 문자열의 모든 줄에서 찾음

정규표현식을 사용하는 클래스

Pattern 클래스

  • 메소드/ 설명

    • public static Pattern compile(String regex) : Pattern 객체를 생성
      • public Matcher matcher(CharSequence input) : 입력을 분석하는 Matcher 객체 생성
      • public static boolean matches(String regex, CharSequence input) : 입력이 regexp에 해당하는지 판단

Matcher 클래스

메소드/ 설명

  • find() : 정규 표현식에 부합되는 내용이 문자열에 포함되어 있는지 반환. 이전 검색 위치부터 이어서 검색.
  • start() : 패턴에 부합되는 요소의 시작 인덱스 반환
  • end() : 패턴에 부합되는 요소가 끝나는 위치 +1을 반환
  • matches() : 문자열 전체가 정규표현식에 일치하는 지를 반환
  • lookingAt() : 비교하려는 문자열이 정규표현식으로 시작하는지 반환. 0번 인덱스부터 검색.
  • replaceFirst() : 일치하는 첫 파턴을 문자열로 대체
  • replaceAll() : 일치하는 모든 패턴을 문자열로 대체
  • reset() : Matcher 의 정보를 리셋하여 0번 인덱스 부터 다시 검색
