Polymorphism

    네 그래요 오늘은 다형성에 대해 알아볼 거십니다.

Override

    뜬금 없지만 제가 좋아하는 노래중 하나는 바로 JP SAXE의 If the world was ending 인데요, 처음에 그 노래를 듣고 검색하려고 구글에 override라고 쳤다가 도저히 안나왔던 기억이 있습니다. 알고보니 'over right' 였더군요. 그냥 그랬다구요 이제 들어가 보죠 #      쟈 다형성을 형성하는 방법으로는 첫번째로 override가 있습니다. override는 상속이 이루어진 경우에만 가능한데요 예를 들어보죠 
class Foo {
               static public String y = "Super Class";
               public String x = "Super";

               public void methodA() {
                   System.out.println("Super Method");
               }
           }

           class Bar extends Foo {
               static public String y = "Sub Class";
               public String x = "Sub";

               @Override
               public void methodA() {
                   System.out.println("Sub Method");
               }
           }

    쟈 이경우는 Bar가 Foo를 상속했습니다. 그리고 methodA()를 Bar에서 오버라이드 하고있죠
지금은 Foo를 상속한 Bar가 하나지만 Bar1,Bar2,Bar3…이렇게 여러개가 있다면 어떨까요? 맞습니다 필요한 자식 클래스로 필요
할때마다 캐스팅 할 수 있겠죠 이렇게 오버라이딩을 통해 다형성을 실현할 수 있습니다.

Overload


    네 두번째 방법은 오버로드인데요 오버로드는 상속이 필요없죠, 그저 필요한 메소드를
상황에 맞게 바꿔쓰시면 됩니다. 이것도 예를 들어보죠

     public Vector3D add(Vector3D x) {
                    return new Vector3D(this.x + x.x,
                            this.y + x.y,
                            this.z + x.z);
                }

                public Vector3D add(float x) {
                    return new Vector3D(this.x + x,
                            this.y + x,
                            this.z + x);
                }

     무섭게 생겼죠? 그래요 나도 무서워요 벡터는 참 못생긴것 같아요 하지만 뜯어보면 이쁜구석이
조금이라도 있을지 모르니 같이 보기로해요. 보면 같은 반환형(Vector3D)에 같은 메소드이름(add)를 가졌네요 그런데 다른게
눈에 띄어요, 바로 파라미터값과 안에 내용이 미묘하게 조큼 다르죠? 그래요 이게 바로 오버로드에요 저그인구수 +8해주는 오버로드가
생각나는 부분이지만 넘어가기로 해요. 결론적으로 오버로드는 파라미터 자료형 또는 갯수가 다를 때 내용을 다르게만들어서 필요할때
바꿔쓸 수 있답니다. 이를 통해서 다형성을 실현할 수 있지요

     public Vector3D add(Vector3D x) {
                     return new Vector3D(this.x + x.x,
                             this.y + x.y,
                             this.z + x.z);
                 }

                 public Vector3D add(float x) {
                     return new Vector3D(this.x + x,
                             this.y + x,
                             this.z + x);
                 }

AbstractClass

    세번째 방법도있어요 세번쨰는 추상클래스를 통한 다형성의 실현이에요 더럽게 종류도 많네요 하지만 여러분과 저는 취업을 해야하니 한번 같이 뜯어보도록 하죠. 저는 예시덕후라 또 예시 한번 가실게요 
abstract class AbstractFoo {
                    int x, y;
                    public void method() {
                        System.out.println("method");
                    }
                    public abstract void abstractMethod();
                }
                class Foo extends AbstractFoo {
                    @Override
                    public void abstractMethod() {
                        System.out.println("implemented abstractMethod");
                    }
                }}
무섭게 생겼네요 하지만 해야지 어쩌겠어요 보시면 abstractMethod();를 포함하고있는 클래스는 지금까지 우리가 보아온 녀석과 좀달라요. 주황버섯에서 뿔버섯이 됫달까요 그런느낌이죠 다들하시죠? abstract class AbstractFoo라고 앞에 abstract가 붙어 있어요 무슨뜻일까요? 네 잘알고 계시네요 추상이란 뜻이에요 추사랑아빠 추상 아니에요 그래서 이 추상 클래스, 추상메소드가 뭘 하느냐? 예를들면 가이드라인 같은거에요. 우리 같이 눈을감고 생각해보죠 당신은 공사장에서 현장안전관리요원으로 취직했어요 그리고 하루 공사가 끝날 때 꼭 체크해야 할 항목들이 있죠. 예를들면 * 개미굴이 몇개인지 * 두더지는 보이지않는지 * 비둘기는 몇kg정도인지 bsp;   체크하는 것들일 거에요 만약에 하나라도 체크를 안하고 퇴근한다면 공사장에 재앙이 생기기 때문에 이것은 회사에서 필수항목으로 체크하라고 가이드라인에 넣어놨다 해보죠. 이게바로 추상메소드입니다. 추상클래스에 추상메소드로 선언해 놓으면 당신은 그것을 무조건 구현해놔야해요 다시 위의 코드를 보죠 abstract class AbstractFoo 라는 가이드라인에서 public abstract void abstractMethod();항목은 꼭 구현해놓으라고 선언한 것이에요 그렇기 때문에 당신이 추상 클래스인 abstract class AbstractFoo를 상속받아서 새로운 클래스를 정의하고 싶다면 class Foo extends AbstractFoo에서 
 
                @Override
                 public void abstractMethod() {
                   System.out.println("implemented abstractMethod");
                  }
이부분을 구현해야 한다는 소리에요 안그러면 안되요 이게바로 추상 클래스입니다.

How-to-start-Git

2020_07_08 나는 오늘 무었을 배웠는가

1.세팅

2.git init에 대해 배웠습니다

3.git clone에 대해 배웠습니다.

1.세팅

* 메일 설정 
 $git config --global user.name "당신의유저네임"
    요로코롬 깃헙과 연결할 유저 이름을 설정 할 수가 있다는 것을 알아줬으면 하는 바에요 * 이름설정 
  git config --global user.email "당신의메일주소"
    같은 방식으로 메일도 설정할 수가 있습니다. * 에디터 설정 
 git config --global core.editor "vim"
    우리는 텍스트 에디터로서 vim을 쓸겁니다. 요로코롬 븸을 깔아주세요

2.git init

* git init보단 clone을 주로 쓴다 합니다. 하지만 깃의 원리를 배우기 위해 배워보아요 
  $mkdir first-git-repo 
  $cd first-git-repo

     위의 두 과정은 각각 깃 레퍼지토리와 연결할 폴더를 만드는 과정과 그 폴더로 이동하는 과정이에요 그쪽으로 이동해야
뭔가를 할 수 있을거에욘
    다음엔 뭘해야 할까요 맞아요 깃허브에도 새로운 저장공간(레퍼지토리)를 만들고 그 둘을 이어주는 작업이
필요할 것이라고 생각되는 부분이에요 그러면 그 과정을 함께 보시죠

 &git init

    울 통해 우리의 first-git-repo에 github과 연결할 준비를 합니다. 이러면 우리 폴더께서
로컬 레퍼지토리로 사용됨을 기대하며 꽃단장을 마친 겁니다. 이제 로컬 레퍼지토리를 원격 (remote)레퍼지토리와 연결시켜 볼까요
 $ git remote add origin 원격 레퍼지토리 주소

    쟈 위의 코드가 바로 로컬 레퍼지토리와 리모트 레퍼지토리를 이어주는 선죽교입니다.
저 코드를 씀으로써 우리는 이제 진정한 깃인이 된것이에요 이후에 README.md파일을 만드는 작업이 우리를 기다려요
    README.md…왜있는 걸까요 저도아직 잘 모릅니다. 하지만 만들라니 만들어야겠지요.
우선 레퍼지토리를 생성하는 순간에 옵션으로 README.md를 만들 수 있어요 그런데말이죠 참 신기하게도 git init으로 하는
과정에서 README.md를 옵션으로 선택해서 연결하면 오류가 뜬다는 거에요 이유는 구글링 해보시기 바랍니다. 그래서 우리는
만들어줄거에요
$touch README.md
$vi README.md

    쟈 우선 첫번째 코드는 리드미 파일을 만드는 것입니다 두번 째는 우리가 앞서 설치한 vim에디터로 리드미를
수정해줄거에요

    i를 눌러 insert mode를 활성화 한 뒤, 텍스트 편집을 수행하고 esc키를 눌러 normal mode로 빠져나와
:wq 를 입력하여 저장한 뒤 vim을 종료합니다.

    이제 파일들을 준비시키고 올려보겠습니다 업로드 준비과정은

$ git add README.md

이고 만약 여러개라면 여러개 띄어쓰기로 구분해서 쓰시면되고 전부다 올려야한다! 하시면은

$ git add *

요로코롬 해주면 될꺼같아요 마지막으로

$ git commit

해주면은 이제 원격 레포지토리에 뙇! 올라갈것입니다. 가 아니에요 올라가지않아요. 안올라간다구요 진지해요
commit과정은 README.md에 대한 설명을 올려주는 과정입니다 아까 vim으로 편집했던 설명을 넣어주는 겁니다. 이제 진짜로
올려보죠
$git push -u origin master

    무섭게 생겼네요 해당 push는 원격레포에 처음으로 올려줄 때만 해주면 되구요 그 이후는
$git push origin master

    요로코롬 올려주시면됩니다. 이렇게 git init 이 끝낫답니당

3.git clone

     아주 깔꼼하고 간편하고 이쁘고 아름다운 방법이라고 말하고있는것만 같아요 아주간단해요 마찬가지로 레포지토리를 생성할 때 README,LICENSE, README.md, .gitignore파일과 함께 생성해 주면 됩니다. 그런데 LICENSE, .gitignore 무섭게 생겼네요 이름처럼 무시하고싶게 생긴아이들이에요 그런데 반대로 저 뜻은 무시하면 안된다는 뜻이라합니다(???)
  • LICENSE
        라이센스는 어느 오픈소스를 기반으로 만들었는지 알려주는 것이라는데 우리 뉴비들은 그냥 MIT 라이센스 쓰면된다는게
    정설입니다
  • .git ignore
        깃 이그노어는 내가 쓸 운영체제, 개발환경, 프로그래밍언어를 고려해서 쓸모 없는 파일들을 알아서 무시해주는
    스마트한 녀석이라고 배웠어요, 아직 뭐가뭔지모르겠지만 그냥 그렇다합니다. 그러면 우리가 일일히 파일 형식을 지정해줘야할까요? 아뇨
    세상은 그렇게 불편하게 발전하지 않았습니다. 간편한 방법을 알아보죠

[gitignore.io]https://www.toptal.com/developers/gitignore
    열로 가시면 자신의 운영체제,개발환경,프로그래밍 언어를 기입하면 .gitignore내용을 자동으로 만들어 줍니다.
그렇다면 우리가 할것은 우리의 gitignore을 vim으로 복붙하는 것 밖엔 없다는 것이에요! 참으로 좋은세상이 아닐 수 없습니다.

    이제 아까과정과 같습니다. 원격 로컬 연결해줄것이에요 해봅시다

$ git clone 복사한remoteRepo주소

    아 그런데 앞에서 얘기 안한게 있는데 주소는 HTTPS주소 쓰셔야되요 SSH쓰는 법은 구글링 ㄱㄱ
    쟈 이제 원격 로컬 연결했고, 리드미도 만들었고 vim으로 수정까지 다하셧겟죠그럼 add하시고 commit하시고
push 하시면 됩니다. 그런데 여기서 또 clone의 좋은점은 앞의 init에서처럼 -u안붙여도 되요 즉

  $git push origin master

    요로코롬 하시면 됩니다. 이상 깃 헙 강의 1일차였습니다.
